# Coding Standards

Permanent rules for Suraj Research Engine. These rules apply to every phase of
development.

- One Module = One Responsibility.
- No Circular Dependency.
- No Hidden Side Effects.
- No Business Logic in main.py.
- Folder names must explain responsibility.
- File names must explain responsibility.
- Function names must explain action.
- Class names must explain role.
- Use Python type hints.
- Use dataclasses where appropriate.
- Prefer composition over inheritance.
- Root Cause Fixes Only.

## Phase Completion Requirements

Every completed phase requires:

- Testing.
- Git Commit.
- Documentation Update.
